# 开场动画注册绑定与恋爱开始日设计

## 目标

将微信登录、资料确认、情侣邀请/加入和恋爱开始日设置融入首页开场动画，让用户感受到共同空间逐渐建立，而不是经历生硬的注册表单。

## 状态流程

- `solo`：未绑定。单人轨迹停留，提供“邀请 TA 来这里”和“我收到了邀请”。
- `profile`：执行邀请或加入前，柔性确认昵称和头像；两项均允许跳过，默认显示“我 / TA”与光点。
- `inviting`：已发出邀请。自己的轨迹点亮半环，远端轨迹呼吸，显示邀请码、复制入口与刷新状态。
- `joining`：受邀方在动画内输入邀请码，绑定成功后直接进入汇合动画。
- `just-connected`：首次绑定成功时播放完整汇合动画。
- `choose-start-date`：汇合后在动画内选择恋爱开始日，可稍后设置。
- `story`：已绑定并设置日期，播放日常故事动画并展示动态恋爱天数。

## 交互原则

- 只有需要操作的动画状态会停留；日常故事动画自动结束。
- 登录由“邀请 TA”或“沿着轨迹去见 TA”触发，不展示 openid、coupleId 等技术概念。
- 昵称和头像均允许跳过；缺失时展示“我 / TA”和光点。
- 发出邀请后再次进入只展示简短等待状态，不阻塞浏览。
- 首页不再显示 mock 恋爱天数；未绑定时显示共同空间引导，已绑定未设置日期时显示温和补充入口。
- 双方均可设置和修改恋爱开始日。

## 数据模型

在 `couples` 记录增加：

```js
{
  relationshipStartDate: '2024-11-20',
  relationshipStartDateUpdatedAt: '2026-06-12T00:00:00.000Z',
  relationshipStartDateUpdatedBy: 'usr_xxx',
  boundAt: '2026-06-12T00:00:00.000Z'
}
```

`getCoupleBindingStatus` 返回共享开始日与绑定时间。新增 `updateRelationshipStartDate` 云函数，通过当前用户 openid 校验情侣归属后更新日期。

## 错误与降级

- 登录、邀请、绑定或日期保存失败时，动画保持当前可操作状态并显示温和 Toast。
- 云端状态读取失败时，不伪造绑定或恋爱天数，保留本地最近状态。
- 日期仅允许合法的 `YYYY-MM-DD` 且不能晚于今天。

## 验证

- 纯逻辑测试覆盖日期合法性、恋爱天数计算和动画模式选择。
- 对所有客户端与云函数 JavaScript 执行 `node --check`。
- 在微信开发者工具中验证完整双账号流程与动画表现。
