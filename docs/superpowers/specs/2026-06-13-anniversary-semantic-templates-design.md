# 纪念日语义模板与重复选项简化设计

## 目标

让系统可靠区分生日和普通纪念日，从而生成更自然的时间文案，同时隐藏用户几乎不需要操作的“每年重复”选项。

## 创建与编辑

纪念日名称下方提供轻量快捷模板：

- 恋爱纪念日
- 生日
- 第一次见面
- 自定义

模板不是传统表单类型下拉框。点击模板会设置内部 `type`，并在名称为空时填入推荐名称；用户之后仍可自由修改名称。

内部 `type` 使用以下稳定值：

- `relationship`
- `birthday`
- `shared_memory`
- `custom`

新建纪念日固定保存为 `repeatType: yearly`，创建表单不再展示重复选择。

编辑已有纪念日时：

- 已有 `repeatType` 保持不变，避免旧的一次性记录被静默修改。
- 旧中文类型会映射到新语义类型。
- 旧“生日”类型统一映射为 `birthday`。
- 系统不判断生日属于哪一方，双方通过纪念日名称表达，例如“阿眠的生日”。

## 展示文案

所有纪念日主信息保持“距离下一次”的倒数，便于用户采取行动。

- `birthday`：辅助显示“已经来到世界 N 天”。
- `relationship`：辅助显示“我们已经一起走过 N 天”。
- `shared_memory`、`custom`：辅助显示“这个日子已经过去 N 天”。

生日不显示“第 N 次纪念”。普通每年纪念日继续显示“这是我们一起纪念的第 N 次”。

列表和详情页均不再显示“每年重复”字样；旧一次性记录仍显示“一次性”，用于解释其不会计算下一年日期。

## 数据与兼容

继续复用现有 `type` 与 `repeatType` 字段，不新增数据库字段或云函数。

客户端和 `upsertAnniversary` 云函数允许新语义类型字符串。旧中文类型继续可读，并在下一次编辑保存时迁移为新值。

日期计算与语义展示放在 `utils/anniversary.js`，页面只消费生成后的文案。

## 测试

- 新建默认每年重复，表单不出现重复选择。
- 生日模板使用中性语义，具体对象由纪念日名称表达。
- 生日、恋爱纪念日和自定义纪念日生成不同辅助文案。
- 生日不显示“第 N 次纪念”。
- 旧中文类型和旧一次性数据继续兼容。
