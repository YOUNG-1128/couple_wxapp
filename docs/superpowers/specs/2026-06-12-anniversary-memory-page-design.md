# 纪念日回忆页优化设计

## 目标

把纪念日从单纯的日期提醒，优化为一个简洁、可持续生长的专属回忆入口。

本阶段交付：

- 纪念日列表更容易浏览，区分即将到来和已经过去的日期。
- 每年重复的纪念日显示这是两人一起纪念的第几次。
- 详情页缩短头图，减少重复信息，让回忆时间线成为主要内容。
- 详情页可直接更换照片背景、改用预设颜色背景或移除照片。
- 详情页提供“记录新的回忆”快捷入口，可进入发动态或写信。
- 保留现有按相同月日自动聚合动态和信件的时间线。

## 不在本阶段

- 不新增纪念日提醒或订阅消息。
- 不新增动态、信件与纪念日的手动关联字段。
- 不新增独立的纪念日照片库。
- 不自动删除被替换后可能无引用的云存储图片。

这些能力可以在当前体验稳定后继续迭代，避免新增纪念日和查看详情的主流程变重。

## 列表设计

列表继续保留“下一个重要日子”的重点卡片，下方内容分为：

- `即将到来`：每年重复的纪念日，以及日期尚未到达的一次性纪念日。
- `已经过去`：日期已经过去的一次性纪念日。

普通列表卡片只承担浏览和进入详情的职责，不直接展示编辑、删除按钮。编辑和删除继续通过详情页或现有编辑入口完成。

每年重复的纪念日显示：

- 倒计时，例如“就是今天”“还有 18 天”。
- 当前将迎来的纪念次数，例如“这是我们一起纪念的第 4 次”。

一次性且已经过去的纪念日显示“已经过去 N 天”，不出现负数倒计时，也不显示纪念次数。

## 详情页设计

详情页头部使用紧凑头图，展示：

- 纪念日名称。
- 倒计时或已经过去的天数。
- 日期与重复方式。
- 每年重复时显示一起纪念的次数。

头图右上角提供半透明的背景操作按钮：

- 有照片时显示“更换背景”。
- 没有照片时显示“设置背景”。

点击按钮打开轻量背景面板，提供：

- 从相册选择照片。
- 预设颜色背景。
- 当前有照片时可移除照片，恢复当前选中的颜色背景。

点击头图的其他区域仍用于预览当前照片。背景按钮需要阻止事件冒泡，避免同时触发预览。

头图下方提供“记录新的回忆”卡片，包含：

- `发动态`：进入现有动态编辑页。
- `写封信`：进入现有写信页。

本阶段快捷入口只缩短创作路径，不写入新的关联字段。创作内容若发生在相同月日，仍由现有自动时间线聚合。

## 数据与架构

### 日期展示

在 `utils/anniversary.js` 中增加纯函数能力：

- 生成正确的倒计时文案，包括过去日期。
- 计算每年重复纪念日当前将迎来的纪念次数。
- 将纪念日分为即将到来和已经过去两组。

页面和服务层复用这些结果，不各自重新计算日期规则。

纪念次数按纪念日原始年份和下一次发生年份计算：

`下一次发生年份 - 原始年份 + 1`

仅每年重复的纪念日显示该值。

### 背景保存

背景更换继续复用现有纪念日记录字段：

- `coverImage`
- `backgroundColor`

选择本地照片后，详情页先通过 `services/cloud-storage.js` 上传，得到可跨设备访问的地址，再调用 `services/anniversary.js` 的现有保存方法。颜色背景直接保存颜色值，并清空 `coverImage`。

详情页只处理选择、面板开关和加载状态；上传、云端同步与本地回退继续复用现有服务约定。

本阶段不需要修改云数据库字段，也不需要新增云函数。

## 错误处理

- 图片选择取消时不提示错误，也不修改当前背景。
- 上传或保存失败时保留原背景，并显示“背景更新失败，请重试”。
- 保存过程中禁用重复操作并显示加载状态。
- 快捷入口导航失败时使用微信默认导航错误行为，不影响详情数据。
- 纪念日详情不存在时继续显示现有“纪念日不存在”提示。

## 测试与验收

自动测试重点覆盖纯业务规则：

- 今天、未来和过去日期的倒计时文案。
- 每年重复纪念日的纪念次数。
- 即将到来与已经过去的分组。
- 农历纪念日继续使用转换后的下一次日期计算。

页面验收重点：

- 列表不再显示负数倒计时。
- 详情头图明显缩短，且不重复展示同一组信息。
- 背景按钮不会触发图片预览。
- 照片、颜色和移除照片三种背景操作均可保存。
- 上传或保存失败后原背景不丢失。
- 发动态、写信快捷入口可进入现有页面。
- 现有相关回忆时间线继续正常展示。
